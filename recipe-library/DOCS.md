# Recipe Library Add-on — Configuration

This document describes **Home Assistant add-on options** in `config.yaml` and how they map to the backend **environment variables** (same semantics as the project root `.env.example`).

The container entrypoint (`docker/entrypoint.js`) reads **`/data/options.json`** (generated by Home Assistant from these options), then sets `process.env` before starting the Node server. If a key is **not** present in `options.json`, the value falls back to any existing container environment variable, then to the **default** listed below.

---

## Server & storage

| Add-on option | Environment variable | Default | Description |
| --------------- | -------------------- | ------- | ----------- |
| `port` | `PORT` | `8097` | HTTP listen port (also aligns with `ingress_port`). |
| `db_path` | `DB_PATH` | `/data/recipe-library.db` | SQLite database file path. Use a path under `/data` for persistence across restarts. |
| `static_dir` | `STATIC_DIR` | `/app/public` | Built frontend assets (usually leave default in the container image). |
| `upload_dir` | `UPLOAD_DIR` | `/data/uploads` | Uploaded recipe and source images; should live under `/data` for persistence. |

**Legacy key names:** `dbPath`, `uploadDir`, `staticDir` are accepted as aliases in the entrypoint for `db_path`, `upload_dir`, and `static_dir`.

## Image processing

| Add-on option | Environment variable | Default | Description |
| --------------- | -------------------- | ------- | ----------- |
| `image_quality` | `IMAGE_QUALITY` | `80` | WebP quality (1–100) for resized uploads. |
| `image_max_dimension` | `IMAGE_MAX_DIMENSION` | `2400` | Longest side cap for main recipe images (px). |
| `text_image_max_dimension` | `TEXT_IMAGE_MAX_DIMENSION` | `1400` | Longest side for images sent to OpenAI for text extraction. |
| `thumbnail_max_dimension` | `THUMBNAIL_MAX_DIMENSION` | `600` | Thumbnail longest side (px). |

## Recipe URL fetch

Used by `POST /api/recipes/extract-from-url` and related flows.

| Add-on option | Environment variable | Default | Description |
| --------------- | -------------------- | ------- | ----------- |
| `recipe_url_fetch_timeout_ms` | `RECIPE_URL_FETCH_TIMEOUT_MS` | `25000` | HTTP fetch timeout (ms). |
| `recipe_url_max_bytes` | `RECIPE_URL_MAX_BYTES` | `2000000` | Maximum response size (bytes). |
| `recipe_url_user_agent` | `RECIPE_URL_USER_AGENT` | `RecipeLibrary/1.0` | User-Agent string for outbound HTTP requests. |

## Naming convention

- **Add-on options** use **snake_case** in `config.yaml` (e.g. `openai_api_key`).
- **Environment variables** use **UPPER_SNAKE_CASE** (e.g. `OPENAI_API_KEY`).
- The entrypoint maps each row in the tables above; see `OPTION_MAP` in `docker/entrypoint.js` for the canonical list.

---

## Persistence & backup

- Mount or use the add-on **`/data`** volume for `db_path` and `upload_dir` so recipes and files survive updates and restarts.
- Back up `recipe-library.db` and the uploads directory together for a consistent restore.
